# 代码与工件治理

## 约束规则

所有代码输出必须符合以下标准：

- 附带简洁注释（关键逻辑说明）
- 附带单元测试示例
- 遵循 Git 工作流：先生成 PR 描述，再输出代码
- 符合团队代码规范（ESLint/Prettier）

## PR 描述模板

```markdown
## 变更摘要
[一句话说明本次变更的目的]

## 变更内容
- [ ] 新增功能：XXX
- [ ] 修复缺陷：XXX
- [ ] 重构优化：XXX

## 测试覆盖
- [ ] 单元测试：XXX
- [ ] 集成测试：XXX

## 风险评估
- 影响范围：[模块/文件列表]
- 风险等级：低/中/高
- 回滚方案：[具体步骤]

## 验收标准
- [ ] 功能正常运行
- [ ] 测试全部通过
- [ ] 无性能退化
```

## 执行要求

- ✅ 先生成 PR 描述，再输出代码
- ✅ 代码必须包含注释和测试
- ✅ 遵循现有项目的代码风格
- ❌ 禁止输出无注释的代码
- ❌ 禁止跳过测试编写
