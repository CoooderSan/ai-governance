# 语雀知识库使用规范

## 概述

语雀（Yuque）是团队的业务知识库，存储架构设计、需求文档、业务流程等内容。
AI 在开发时应主动搜索相关文档，作为理解需求的上下文。

---

## 前置配置要求

### 1. 安装 yuque-dl

```bash
npm install -g yuque-dl
```

### 2. 配置个人 Token

在 `~/.claude/settings.json` 中配置：

```json
{
  "yuque": {
    "token": "your-personal-token",
    "repo": "https://www.yuque.com/org/repo",
    "cacheDir": "~/.cache/yuque"
  }
}
```

**获取 Token**：https://www.yuque.com/settings/tokens

**重要**：
- ⚠️ 使用你自己的 token，不要共享
- ⚠️ 不要把 token 提交到 Git
- ⚠️ Token 继承你的所有权限

### 3. 同步文档

**方式 1：使用 Skill（推荐）**

直接告诉 AI：
```
"同步语雀文档"
或
"/trellis:sync-yuque"
```

AI 会自动调用 `yuque-dl` 同步文档。

**方式 2：手动同步**

```bash
npx yuque-dl <repo-url> -t <token> -d ~/.cache/yuque --incremental
```

**方式 3：使用脚本**

```bash
node .agents/skills/sync-yuque/scripts/sync-yuque.js
```

## 使用时机

AI 应在以下情况主动搜索语雀文档：

| 场景 | 说明 | 示例 |
|------|------|------|
| **收到开发需求** | 需要理解业务背景 | "实现 WES 拣货流程" |
| **不确定需求细节** | 需要查找设计文档 | "AMS 系统的认证方式是什么？" |
| **跨系统集成** | 需要了解接口定义 | "对接 WMS 系统" |
| **架构决策** | 需要参考现有设计 | "如何设计库存模块？" |

---

## 搜索策略

### 第一步：提取关键词

从用户需求中提取业务关键词：

**示例**：
- 需求："实现 WES 拣货流程的异常处理"
- 关键词：`WES`, `拣货`, `异常处理`

### 第二步：搜索文档标题

使用 Grep 工具搜索 `~/.cache/yuque/` 目录：

```bash
# 搜索包含关键词的文档
grep -r "WES" ~/.cache/yuque/ --include="*.md" -l
grep -r "拣货" ~/.cache/yuque/ --include="*.md" -l
```

### 第三步：过滤和排序

按文档类型优先级排序：

| 优先级 | 文档类型 | 关键词 |
|--------|---------|--------|
| 🔴 **高** | 架构设计、技术方案 | "架构"、"设计"、"方案" |
| 🟡 **中** | 业务流程、接口文档 | "流程"、"接口"、"API" |
| 🟢 **低** | 操作手册、FAQ | "手册"、"FAQ"、"指南" |

### 第四步：读取文档

选择最相关的 **2-3 个文档**读取：

```bash
# 读取文档内容
cat ~/.cache/yuque/WES架构设计.md
cat ~/.cache/yuque/拣货流程说明.md
```

**重要**：不要读取过多文档，避免上下文过载。

## 执行要求

### ✅ 必须做

1. **主动搜索**：收到开发需求时，主动搜索语雀文档，不要等用户要求
2. **提取关键词**：从需求中提取 2-5 个业务关键词
3. **限制数量**：最多读取 3 个文档，避免上下文过载
4. **说明相关性**：告诉用户为什么选择这些文档
5. **摘要呈现**：不要全文粘贴，提取关键信息

### ❌ 禁止做

1. **禁止跳过搜索**：即使觉得不需要，也要尝试搜索
2. **禁止读取过多**：超过 3 个文档会导致上下文混乱
3. **禁止全文粘贴**：长文档应提取摘要
4. **禁止假设内容**：如果没找到文档，明确告知用户

## 错误处理

### 情况 1：语雀缓存不存在

```
⚠️ 语雀缓存目录不存在

未找到 ~/.cache/yuque/ 目录。

请先配置语雀文档同步：
1. 安装 yuque-dl: npm install -g yuque-dl
2. 同步文档: yuque-dl <repo> -t <token> -d ~/.cache/yuque
3. 或配置 yuque-ecosystem MCP

配置完成后，我可以自动搜索相关文档。
```

### 情况 2：未找到相关文档

```
📚 语雀搜索结果：未找到相关文档

搜索关键词：WES, 拣货
搜索路径：~/.cache/yuque/

可能原因：
- 文档尚未同步到本地
- 关键词不匹配
- 文档使用了不同的命名

建议：
1. 手动检查语雀知识库
2. 提供更多关键词
3. 重新同步语雀文档
```

### 情况 3：文档过多

```
📚 找到 15 个相关文档

由于文档数量较多，我将只读取最相关的 3 个：
1. WES架构设计.md（匹配度：高）
2. 拣货流程v2.md（匹配度：高）
3. 异常处理规范.md（匹配度：中）

如需查看其他文档，请告诉我具体文档名。
```

```
用户："实现拣货功能，参考 @yuque:拣货流程v2.0"
AI：直接读取 ~/.cache/yuque/拣货流程v2.0.md
```

## 最佳实践

### 1. 搜索时机

```
✅ 好的做法：
用户："实现 WES 拣货功能"
AI："让我先搜索语雀中的 WES 和拣货相关文档..."

❌ 不好的做法：
用户："实现 WES 拣货功能"
AI："好的，我开始写代码..."（没有搜索文档）
```

### 2. 关键词提取

```
✅ 好的做法：
需求："优化 AMS 系统的库存同步性能"
关键词：AMS, 库存, 同步, 性能

❌ 不好的做法：
需求："优化 AMS 系统的库存同步性能"
关键词：优化, 系统（太泛化）
```

### 3. 文档呈现

```
✅ 好的做法：
"找到《WES架构设计》，其中提到拣货模块使用消息队列..."

❌ 不好的做法：
"找到文档，内容如下：[粘贴 5000 字全文]"
```

## 与 Intent Gate 的配合

语雀搜索应在 Intent 文档检查**之前**执行：

```
流程：
1. 收到开发需求
2. 搜索语雀文档（本规范）
3. 检查 Intent 文档（intent-gate.md）
4. 如果没有 Intent，引导用户创建（并提供语雀文档作为参考）
```

## 配置示例

用户可以在 `~/.claude/settings.json` 中配置语雀：

```json
{
  "yuque": {
    "cacheDir": "~/.cache/yuque",
    "maxDocs": 3,
    "autoSearch": true,
    "keywords": ["WES", "AMS", "WMS"]
  }
}
```

**说明**：
- `cacheDir`: 语雀缓存目录
- `maxDocs`: 最多读取文档数量
- `autoSearch`: 是否自动搜索
- `keywords`: 项目常用关键词（可选）

---

## 总结

**核心原则**：
1. 主动搜索，不要等用户要求
2. 精准过滤，只读取相关文档
3. 摘要呈现，避免上下文过载
4. 明确说明，让用户知道你在做什么

**记住**：语雀文档是理解业务需求的关键，但不要让它淹没代码上下文。
